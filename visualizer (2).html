What is Forwarding?

In a 5-stage pipeline (IF → ID → EX → MEM → WB), a RAW hazard occurs when an instruction needs a register that the previous instruction hasn't written yet. Without forwarding, the pipeline must stall (insert bubbles) until the value is written back to the register file — up to 3 cycles wasted. With forwarding, the result is forwarded directly from the EX or MEM stage output to the EX stage input of the dependent instruction — 0 extra stalls (except load-use which still needs 1 cycle).
